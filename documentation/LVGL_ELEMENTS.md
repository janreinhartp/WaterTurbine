# LVGL Elements and How To Use Them

This file catalogs the LVGL screens/elements currently generated and used in the project.

## 1. UI Framework Context

- UI generated by SquareLine Studio
- LVGL version used by UI files: 9.2.2
- Entry point: `ui_init()` in `main/ui/ui.c`

## 2. Screen List

- `ui_MainMenu`
- `ui_Gauges`
- `ui_Data`
- `ui_Settings`

All screens are initialized during `ui_init()` and Main Menu is loaded first.

## 3. Main Menu Elements (`ui_MainMenu.c`)

Main elements:

- `ui_btnGauges` (button)
- `ui_btnData` (button)
- `ui_btnSettings` (button)
- `ui_logo` (image)
- `ui_Label4` (title text)

Navigation callbacks:

- `ui_event_btnGauges` -> loads `ui_Gauges`
- `ui_event_btnData` -> loads `ui_Data`
- `ui_event_btnSettings` -> loads `ui_Settings`

## 4. Gauges Elements (`ui_Gauges.c`)

Arc widgets:

- `ui_gVoltage`
- `ui_gAmpere`
- `ui_gRPM`
- `ui_gPower`
- `ui_gWaterFlow`

Value labels:

- `ui_gVoltageValue`
- `ui_gAmpereValue`
- `ui_gRpmValue`
- `ui_gPowerValue`
- `ui_gWaterFlowValue`

Buttons:

- `ui_btnGaugeBack` -> Main Menu
- `ui_btnGaugeData` -> Data screen

How to update a gauge:

```c
lv_arc_set_value(ui_gPower, 72);
lv_label_set_text(ui_gPowerValue, "126 W");
```

## 5. Data Elements (`ui_Data.c`)

Chart elements:

- `ui_Chart1` (line chart — converted from bar to line at runtime by `gauges_controller`)
- `ui_Chart1_Xaxis`
- `ui_Chart1_Yaxis1`
- `ui_Chart1_Yaxis2`

Metric label groups:

- Flow: `ui_lblFlowRateValue`
- Voltage: `ui_lblVoltageValue`
- Ampere: `ui_lblAmpereValue`
- Power: `ui_lblPowerValue`
- RPM: `ui_lblRpmValue`

Buttons:

- `ui_btnDataBack` -> Main Menu

How to update labels:

```c
lv_label_set_text(ui_lblFlowRateValue, "6.1 L/M");
lv_label_set_text(ui_lblRpmValue, "950 RPM");
```

## 6. Settings Elements (`ui_Settings.c`)

Editable controls:

- `ui_txtbSettingsCurrentValue` (single-line text area)
- `ui_Keyboard2` (numeric keyboard)

Buttons:

- `ui_btnSettingsSave` (implemented callback: returns Main Menu)
- `ui_btnSettingsNext` (no callback yet)
- `ui_btnSettingsBack` (no callback yet)

Keyboard binding is already configured:

```c
lv_keyboard_set_textarea(ui_Keyboard2, ui_txtbSettingsCurrentValue);
```

## 7. Using Custom Handles (`uic_*`)

Each screen also exports `uic_*` pointers, which mirror the generated `ui_*` objects.

Use whichever naming convention you standardize on, but keep it consistent in your own app logic.

## 8. Screen Transitions

Current transitions use helper API and fade animation, for example:

```c
_ui_screen_change(&ui_Data, LV_SCR_LOAD_ANIM_FADE_ON, 200, 0, &ui_Data_screen_init);
```

To add a new transition:

1. Create the target screen init function.
2. Add an event callback to your button.
3. Call `_ui_screen_change(...)` with target screen and init function.

## 9. Runtime Data Updates (gauges_controller)

The `gauges_controller` module already provides live updates to all metric widgets:

- **Gauge arcs** — `lv_arc_set_value()` with percentage-mapped values every 1 second
- **Gauge labels** — `lv_label_set_text()` with formatted strings (e.g., `"24.3 V"`)
- **Data chart** — `lv_chart_set_next_value()` on 3 line series (FlowRate, Voltage, Ampere) in shift mode
- **Data labels** — `lv_label_set_text()` for all 5 metric panels

All updates run inside a FreeRTOS task and use `lvgl_port_lock()` / `lvgl_port_unlock()` for thread safety.
